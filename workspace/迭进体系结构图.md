## 一、文件体系

```
workspace/
├── AGENTS.md                  # 强制执行规则（初始化/预检/首字符/研判清单）
│   └── 仅保留：迭进初始化、预检执行、首字符规则、研判清单强制标准
│
├── dgen_rules.md              # ⭐ 迭进规则唯一源（由 dgen_evolve.py 自动维护）
│   ├── 13条拦截规则（通用拦截规则）
│   ├── 9条成功模式（通用成功模式）
│   ├── 附录F：上下文分层管理（L1永久/L2摘要/L3临时）
│   ├── 附录G：迭进自动化引擎
│   └── 已生效的自动提议：cron_failover降级 + rule_prioritize排序
│
├── 工作手册.md
│
├── MEMORY.md                  # 长期记忆（状态/版本/关键决策）
│
├── state.json           # 唯一状态源（运行状态/关键数据）
│
├── context.json          # 最新轮询快照（L3临时数据）
│
├── scripts/
│   └── dgen_evolve.py         # ⭐ 迭进自动化引擎
│       ├── 自动记录观察 → rule_health.json
│       ├── 自动生成提议 → pending_review
│       ├── 自动验证门控 → 你确认生效
│       └── 仅操作 dgen_rules.md
│
├── memory/
│   ├── rule_health.json       # 规则健康度 + 观察/提议记录
│   ├── failures.json          # 失败快照（最近20条）
│   ├── trail_YYYY-MM-DD.md    # 执行轨迹（每日关键决策推理链）
│   └── YYYY-MM-DD.md          # 每日日志
```

## 二、执行流程

```
用户发消息
  → DGEN预检（dgen_pre_check 工具）
    → 拦截？→ 输出拦截原因
    → 通过？→ [DGEN] ✅ 通过
      → 工作流程（①数据采集→②分析→③判断→④决策→⑤执行）
        → 关键操作？
          → 写入 state.json
          → 写入 memory/trail_*.md
          → 触发 dgen_evolve.py：
            → 自动观察 → 规则健康度异常？
              → 自动生成提议 → pending_review
              → 你确认？→ 写入 dgen_rules.md
```

## 三、三层管理

| 层级 | 内容 | 裁剪策略 | 存放位置 |
|:---|:---|:---|:---|
| L1 永久 | 状态/阈值/数据 | 不可裁剪 | state.json |
| L1 永久 | 规则/模式/自检流程 | 不可裁剪 | dgen_rules.md |
| L2 摘要 | 运行参数/每日简报 | 可摘要化 | SYSTEM_CONFIG.md + memory/YYYY-MM-DD.md |
| L2 摘要 | 执行轨迹 | 可摘要化 | memory/trail_*.md |
| L3 临时 | 原始数据/任务快照 | 可丢弃 | 重新读取即可 |
