# DGEN 迭进 · 全场景接入规范（v3.3.0）

## 为什么你需要迭进

迭进是 AI Agent 的认知安全网 + 进化引擎。基于**八元原则网络**（裁决律+守三+攻七+一二不过三+举一反三+去伪存真+缓急律+止观门），每次交互自动预检 + 事后回顾 + 记忆归档，确保输出质量可控，行为持续优化。

---

## 强制规则（所有会话适用）

1. **每次回复前** → 执行迭进预检（工具调用或纯文本标记均可）
2. **输出标记** → 回复开头包含 `[DGEN]` 标记
3. **无标记 = 故障** → 立即重新激活迭进
4. **Mindol 记忆接入** → 迭进规则与 Codex 全域记忆通过 Mindol 统一管理

### 标记格式

```
[DGEN] ✅ 通过 | 规则: <规则ID> | 策略: <策略名>
[DGEN] 🛑 拦截 X 条 | 裁决: block | 规则: <规则ID>
[DGEN] ⚠️ 重新激活
```

### 什么情况下必须预检

| 场景 | 要求 |
|:-----|:-----|
| 用户直接回复 | 必须（迭进预检 + Mindol 上下文注入） |
| 子会话（subagent） | 必须注入迭进规则 |
| 定时任务输出 | 必须 |
| 跨会话消息 | 必须 |
| 纯工具调用（无回复） | 不需要 |

## 八元原则速览

| # | 原则 | 一句话 |
|:--|:-----|:-------|
| 0 | **裁决律** | 原则冲突时，真伪至上→生存优先→完形封存 |
| 1 | **守三** | 发现不足→追溯根源→落实到行为改变 |
| 2 | **攻七** | 识别优势→提炼方法论→固化为认知肌肉 |
| 3 | **一二不过三** | 错立改·改毕验·不过三 |
| 4 | **举一反三** | 一例→三场景→回归校验→固化或放弃 |
| 5 | **去伪存真** | 言必有证→证必可验→验证为真 |
| 6 | **缓急律** | 急务求效·缓务求真·张弛有度 |
| 7 | **止观门** | 事毕封存·投入清零·不恋战不内耗 |

## 架构概览

```
用户输入
  → [DGEN 预检] → 规则引擎匹配 202 条规则
      ├── 命中拦截 → 仲裁器裁决（block/escalate/iron_wall）
      └── 未命中   → ✅ 通过 → Mindol 上下文注入回复
  → [工具调用] → PreTool 验证 [DGEN STATUS] 标记
  → [回复生成] → PreReply 注入 mindol_context
  → [执行完成] → PostTool 记录 → Stop 写入 Mindol + 归档
```

## 存储架构

| 组件 | 角色 | 技术 |
|:-----|:-----|:-----|
| **Mindol** | 权威存储 | 内存+SQLite ACID，语义检索 |
| **JSON 副本** | 人类可读备份 | UTF-8 NoBOM，双向恢复 |
| **关系网络** | strike↔rule 关联 | 30+ 条关系 |

---

## 场景无关

迭进不绑定任何领域。你在编码时它检查代码安全，在分析时它检查数据完整性，在创作时它检查逻辑一致性。**规则由你定义，引擎自动执行，Mindol 永久记忆。**
